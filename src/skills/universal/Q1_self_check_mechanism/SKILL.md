## 说明

检查复杂任务的 SP 是否设计了输出前自我检查环节，以及该环节是否具备
实际约束力（有具体检查清单，而非泛泛而谈）。

## 执行模式说明

需理解整体任务逻辑判断自检环节是否真正有效，标记为 llm_judge。

## 检查项

### Q1-1 完全没有设计自我检查环节
检查：任务涉及多步骤/多字段要求，但全篇没有要求模型输出前自检。
默认 severity：major
fix 模板：
{ "action": "text_insert", "target": "<输出要求段落末尾>",
  "content": "补充：在正式输出前自己检查一遍：是否包含所有必填
  字段、格式是否符合要求、内容是否有明显逻辑矛盾。如发现问题，先
  修正后再输出" }

### Q1-2 自检标准过于笼统
检查：提到了"输出前检查一下"，但检查标准笼统，缺乏具体清单。
默认 severity：minor
fix 模板：
{ "action": "text_replace", "target": "<笼统的自检描述>",
  "from": "<原笼统描述>", "to": "<具体检查清单，例如：1.是否包含
  全部必填字段；2.字数是否符合限制；3.是否有前后矛盾内容>" }

### Q1-3 发现问题后处理方式未说明
检查：设计了自检环节和具体标准，但未说明发现问题后该怎么处理。
默认 severity：minor
fix 模板：
{ "action": "text_insert", "target": "<自检环节描述末尾>",
  "content": "如果自我检查发现问题，重新调整后再输出最终结果，
  不要在未解决问题的情况下直接输出" }

## Golden Set

样本1（应判 fail，major）："根据用户需求，生成一份包含标题、正文、
标签三个部分的文章。"
样本2（应判 pass）："……在正式输出前，自己检查：1.三个部分是否都已
完整生成；2.标题是否在20字以内；3.标签是否为3到5个关键词。如发现
任何一项不符合，先调整后再输出最终结果。"
样本3（应判 fail，minor）："生成文章后，自己检查一下有没有问题再
输出。"
