## 说明

检查 SP 是否对"模型编造看似合理但无依据内容"的风险做出限制，尤其在
涉及精确事实、法律/医疗/金融等高风险领域时。

## 执行模式说明

需结合任务场景判断风险程度，标记为 llm_judge。

## 检查项

### R5-1 涉及精确事实但未要求标明不确定性
检查：任务涉及需要精确、可核实的事实（法律条款、产品参数等），但未
要求模型在不确定时明确说明。
默认 severity：critical（高风险领域）/ major（一般事实场景）
fix 模板：
{ "action": "text_insert", "target": "<相关任务描述处>",
  "content": "补充说明：如果对某个具体细节没有确切依据，明确说明
  '这一点我不确定，建议核实'，不要编造看似合理但无依据的细节" }

### R5-2 未说明信息不足时如何处理
检查：任务需依赖用户提供的具体信息，但未说明信息缺失/不足时该如何
应对。
默认 severity：major
fix 模板：
{ "action": "text_insert", "target": "<依赖信息的任务描述处>",
  "content": "只根据用户提供的信息进行分析，不要假设或编造未提供
  的数据；信息不足时明确说明还需要哪些信息" }

### R5-3 要求输出引用/来源/数字但未要求核实依据
检查：要求输出具体引用来源、统计数字，但未要求这些内容必须有真实
依据。
默认 severity：critical
fix 模板：
{ "action": "text_insert", "target": "<引用要求处>",
  "content": "补充说明：所有引用来源、数字、文献必须有真实依据；
  无法确认真实性时不要输出，而是说明需要人工核实" }

## Golden Set

样本1（应判 fail，critical）："回答用户关于劳动合同法的具体问题，
给出准确的法律条款依据。"
样本2（应判 pass）："回答用户关于劳动合同法的一般性问题……如果对
具体条款细节没有十足把握，明确说明'建议咨询专业律师核实'，不要
编造不确定的具体法条内容。"
样本3（应判 fail，major）："根据用户上传的销售数据，分析销售趋势
并给出结论。"（未说明数据不完整或缺失时如何处理）
